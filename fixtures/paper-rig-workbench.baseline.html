Rig package

    PURPOSE
Self-contained paper-rig/1 reference workbench: rig data → local-pose solver → orthographic projection → semantic compositor → JSON/SVG export → validation UI.

SOURCE OF TRUTH
Rig objects and exported JSON. Never scrape rendered SVG or UI state. Preserve stable model, joint, plate, gasket, paint, and anchor IDs.

COORDINATES
Meters. +X forward, +Y model-left, +Z up. Ground Z=0. Origin centered between contacts. Left/right are absolute; camera decides near/far.

EDIT RULES
Rotate parent joints to move complete chains. Do not translate endpoints to fake reach. Keep planted contacts on Z=0. Use under-core for appendages that originate beneath a mass; surface-relative for horns/tusks/details that can cross a surface. Keep all geometry closed, opaque, role-colored, attached, and gasketed.

REQUIRED CHECKS
Syntax. Package validation passed. Bind/idle/walk/attack/hit/ko key poses. Eight headings × 30°/45°/60°/75°. Inspect animation extremes at 60° heading 0° and the reverse heading. Verify contacts, fixed bone lengths where declared, core occlusion, bilateral swaps, stable IDs, anchors, opacity, and far LOD.

SCALE TO 100 MODELS
Treat this HTML as reference/demo, not the authoring database. Split compiler/runtime, schema, validator, workbench UI, family presets, and one rig JSON per model. Run a CLI validator/contact-sheet generator in CI. Keep browser workbench as a viewer/editor over imported packages.
